# 專案初始化與目錄規範

日期：2026-06-26 14:02

## 專案目標
將 **OpenTyrian**（原始 C 語言，源自 Turbo Pascal 的 DOS 遊戲 Tyrian 移植版）
移植為 **.NET 10 / C#** 版本。

## 目錄結構
| 目錄 | 用途 |
|------|------|
| `sources/` | 原始 C 原始碼（clone 自 github.com/opentyrian/opentyrian），**唯讀僅供對照** |
| `cs_ported/` | 移植後的 C# 專案原始碼 |
| `Build/` | 移植建置產出（`dotnet build -o`） |
| `MD/` | 各類文件，依性質分子目錄 |
| `MD/Note/` | 重要筆記，檔名 `年月日時分-主題.md` |
| `private/` | 私人/暫存，不納入移植流程 |

## 環境
- .NET 10 SDK 已安裝：`10.0.109`
- 平台：Windows 11
- 原始碼 ~50 個 `.c` 模組 + 對應 `.h`，相依 SDL2 + SDL2_net、OPL 音源、VGA 256 色

## 待決事項

2. 專案型態（單一 console vs 多專案 solution）
3. 網路多人是否納入首版

## 守則重點
- 不修改 `sources/`；逐模組對照移植，行為一致優先
- Pascal 型別對應：`JE_longint`→`int`、`JE_word`→`ushort`、`JE_byte`→`byte` 等
- 二進位存檔/設定需對齊型別寬度與 little-endian
