# 架構設計

> 決策記錄見 `MD/Note/202606261410-架構決策-六角形與雙專案.md`。本檔為現行設計的權威說明。

## Ports & Adapters（六角形）
**依賴方向單向：`App` → `Core`；`Core` 絕不認識 SDL。** 換平台只改 Adapter，核心不動。

```
cs_ported/
├─ AprCSTyrian.slnx
├─ Core/   (class lib, 零 SDL)  ── Ports 介面 + 遊戲邏輯
└─ App/    (WinExe, 參考 Core)  ── SDL Adapters + Program.cs 組合根
```

## Core（class library，零 SDL 依賴）
純遊戲邏輯——關卡、敵人、武器、計分、存檔資料、亂數、主迴圈狀態。
只依賴自己定義的 **Ports（介面）**：

| Port | 職責 |
|------|------|
| `IVideoBackend` | 畫面輸出（接收 320×200 indexed 緩衝 + 調色盤） |
| `IAudioBackend` | 音訊輸出（接收已混好的 PCM；OPL 混音邏輯屬 Core） |
| `IInputBackend` | 輸入事件/狀態（鍵盤、搖桿） |
| `IClock` | 時間/延遲 |
| `IFileSystem` | 資料檔/存檔存取 |
| `IGamePlatform` | 聚合上述全部，供注入 |

## App（WinExe，參考 Core）
- **SDL Adapters**：`SdlVideo` / `SdlAudio` / `SdlInput` / `SdlClock` / `PhysicalFileSystem`，
  各自實作對應 Port；`SdlPlatform` 聚合並管理 SDL 生命週期。
- **`Program.cs` 組合根**：建立 SDL 平台並注入 `TyrianGame`，是唯一知道「Core 跑在 SDL 上」的地方。

## 專案型態（已定案）
- 單一 solution、**雙專案**（`Core` + `App`），皆位於 `cs_ported/`。
- 選雙專案是為了讓**編譯器強制隔離**：Core 不參考 SDL 組件，誤用即編譯失敗。
- **無 console 黑窗**：`App` 設 `<OutputType>WinExe</OutputType>`。
- **視窗**：SDL 開單一視窗，標題列右上 X 關閉，其餘 client 區即遊戲畫面。
- **建置輸出**：單一執行檔輸出至 `Build/`。
- **網路多人**：**不納入**，移植時跳過 `network.c` / SDL2_net。
