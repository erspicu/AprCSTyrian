# 執行步驟規劃（移植路線圖）

## 移植風格（總原則）
**以 C 風格逐函式翻譯為主，盡量只做語法轉換，不重新設計架構。**
- 每個 `.c` → 一個同名 `static class`；C 全域 → `static` 欄位；C 函式 → `static` 方法。
- **保留原始識別字命名**（如 `mt_srand`、`JE_loadPic`、`JE_byte`），方便與 `sources/` 逐行對照。
- **盡量用非託管陣列/指標**（`unsafe`、`NativeMemory`）對應 C 的 `static` 陣列與指標運算；
  全域緩衝在啟動時配置一次、結束時統一釋放，**嚴禁 memory leak**（見 `MD/Reference/移植風格與C慣例.md`）。
- 減少不必要的物件包裝；把整個 GAME LOOP 當成 C 程式去完成。
- Ports 介面（`IVideoBackend` 等）僅作為 SDL 邊界薄接縫；遊戲邏輯不直接碰 SDL。
