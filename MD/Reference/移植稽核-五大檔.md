# 移植稽核：五大核心檔案逐函式比對

逐函式實際比對「原始 C」與「C# 移植」（由稽核代理逐行讀碼產生）。
涵蓋 `tyrian2.c` / `mainint.c` / `game_menu.c` / `opentyr.c` / `menus.c`。

**狀態定義：**

| 標記 | 意義 |
|---|---|
| ✅ 忠實 | 逐行語法翻譯，與原版邏輯一致 |
| 🟡 部分 | 有實作但不完整 / 部分簡化 / 缺大段 |
| 🔴 自創 | **非翻譯**，自行重建的簡化版（bug 來源） |
| ⬜ stub | 占位空殼，無實際邏輯 |
| ❌ 缺 | 未移植 |

> **最關鍵結論**：核心 gameplay 三大件嚴重不完整 ——
> 1. **`JE_eventSystem`：約 75 個事件 case 只移植了 10 個**（缺 ~65，含敵人移動/加速/動畫/生敵變體/關卡結束/文字窗/濾鏡/計時器等）。
> 2. **`JE_main`：自創簡化骨架，缺 19 大段原版邏輯**（濾鏡/分層敵人繪製/玩家碰撞/暫停選單/能量再生/gamma…）。
> 3. **`JE_playerMovement`（1160行）：未移植，以自創簡化骨架代替**。
>
> 商店 `game_menu.c` 與選單 `menus.c` 則幾乎全部忠實移植。

## 總結與優先修正順序

**統計**（101 函式）：✅忠實 ~50、🟡部分 ~9、🔴自創 3、⬜stub ~8、❌缺 ~31。

**已可信完整：** 商店(game_menu.c 全)、New Game 選單(menus.c 全)、標題畫面、敵人工廠/事件生成、存讀檔/高分/說明、過關動畫、main 初始化骨幹。

**gameplay 失準根源（建議依序忠實移植）：**
1. 🔴 **JE_eventSystem 補齊 case 9-81**（約65個）—— 敵人移動/動畫/生敵/關卡流程全靠它，**最優先**。
2. 🔴 **JE_main**（tyrian2.c 624）—— 丟掉自創骨架逐行翻譯，補回19大段（尤其分層敵人繪製、濾鏡、玩家碰撞）。
3. 🔴 **JE_playerMovement + JE_mainGamePlayerFunctions + JE_playerCollide**（mainint.c）—— 玩家移動/射擊/邊界/僚機/特武。
4. 🟡 **JE_drawEnemy** 補 Enemy Launch Routine（發射子機/aim/linknum）。
5. ❌ **JE_mainKeyboardInput / JE_pauseGame / JE_doInGameSetup / JE_inGameSetup** —— 遊戲中輸入/暫停/設定。
6. ❌ **JE_gammaCorrect 三函式** —— JE_main setup 漏的調色盤步驟。
7. ⬜ **setupMenu**（opentyr.c）—— 首頁 SETUP 項目；**newSuperArcade/TyrianGame、JE_whoa、JE_displayText、JE_nextEpisode** 等 stub。
8. 🟡 **JE_itemScreen** 補主選單現金/護甲條 + 搖桿設定渲染。
