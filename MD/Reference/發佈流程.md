# 發佈打包工作流（Release）

把 AprCSTyrian 打包成「點了就能玩」的成品 + 發佈到 GitHub Release。

## 一鍵發佈

```bash
bash tools/release.sh
```

之後若說「更新發佈」，就是再跑這支腳本一次。

## 版號機制

`YYYYMMDD-<git短hash>`，例如 `20260627-7b91d1a`。
- 日期：發佈當天 `date +%Y%m%d`
- 短 hash：`git rev-parse --short HEAD`（對應發佈時的 commit）

壓縮檔命名：`AprCSTyrianRelease-<版號>.zip`（例 `AprCSTyrianRelease-20260627-7b91d1a.zip`）。

## 產出物

| 產物 | 路徑 | 說明 |
|---|---|---|
| 可玩資料夾 | `AprCSTyrianRelease/` | 直接點 `AprCSTyrian.exe` 就能玩（已含 runtime + 資料） |
| 發佈壓縮檔 | `AprCSTyrianRelease-<版號>.zip` | 上傳 GitHub Release 的檔案 |
| GitHub Release | tag = `<版號>` | 自動建立，附上壓縮檔 |

兩者皆已列入 `.gitignore`（不進版控）。

## 腳本做了什麼（5 步）

1. **`dotnet publish`**（self-contained `win-x64`，**單檔**）：把所有受管理 DLL、.NET runtime、
   原生 `SDL2.dll` 全部併入**單一 `AprCSTyrian.exe`**（`PublishSingleFile` + `IncludeNativeLibrariesForSelfExtract`），
   **使用者端不需安裝 .NET、也不用散佈一堆 DLL**。輸出到 `AprCSTyrianRelease/`。
   - App.csproj 的 `TrimUnusedSdlNatives` target 只保留 `SDL2.dll`，排除遊戲未用的
     `SDL2_image / SDL2_mixer / SDL2_ttf` 及其編解碼相依（`lib*.dll`），所以併入 exe 的原生庫只有 `SDL2.dll`。
2. **複製遊戲資料**：`Build/data` → `AprCSTyrianRelease/data`（Tyrian 2.1 freeware 資料，點了就能玩）。
3. **寫 `VERSION.txt`**：版號 / 日期 / 完整 commit。
4. **打包 zip**：以 `AprCSTyrianRelease/` 為頂層資料夾壓成 `AprCSTyrianRelease-<版號>.zip`。
5. **建立 GitHub Release**：`gh release create <版號> <zip>`；若同 tag 已存在則先刪除重建。

## 前置需求

- `dotnet`（建置用；發佈出的成品本身不需 .NET）
- `gh` CLI 已登入（`gh auth status` 應為 logged in；本機已登入 `erspicu`）
- `Build/data` 存在（Tyrian 2.1 freeware 資料；無則腳本報錯）
- 乾淨工作區（有未提交變更時腳本會警告，因版號的 commit hash 不含未提交改動）

## 注意

- 成品：`AprCSTyrian.exe`（單檔，約 ~15 MB，已 `PublishTrimmed` 修剪未用的 runtime 程式碼，含 SDL2.dll）+ `data/`（遊戲資料）。執行時會在 exe 同目錄建立 `config/` 放設定/存檔。
  - 註：修剪後若發現某功能異常（理論上本專案為 C 移植、無反射，風險低且已驗證標題/demo/渲染/設定讀寫），可移除 `release.sh` 的 `-p:PublishTrimmed=true` 回到未修剪（~73 MB）。
- 資料為 Tyrian 2.1 **freeware**（2004 年釋出為免費），可隨成品散布。
- 換平台（如 Linux/macOS）：改 `-r` 的 RID（如 `linux-x64`）即可，SDL2 原生庫由 NuGet 帶入。

## KeyLog（重播/截圖）除錯機制開關

`KeyLog.cs` 的 keylog 擷取 / 重播 / 截圖機制（開發/除錯用）由原始碼總開關 `KeyLog.Enabled` 控制：
- **平時開發** = `true`：保留機制，用環境變數啟用（`KEYLOG=1` / `KEYLOG_REPLAY=...` / `KEYLOG_DIR=...` / `KEYLOG_FORCE=1`）。
- **正式發佈** = `false`：`tools/release.sh` 以 `-p:KeyLogOff=true` 編出 `KEYLOG_OFF` 符號自動關閉，發佈版完全 no-op（環境變數也不生效、相關碼被死碼消除），**不需手動改**。
- 想手動關/開：改 `Core/KeyLog.cs` 的 `Enabled` 常數即可。

> 還沒穩定前保留此機制；穩定後可考慮移除。
